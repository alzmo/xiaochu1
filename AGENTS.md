# Codex 工作规范（必须遵守）

## 1. README 更新规则（强制）

每次完成功能修改或结构调整后，必须同步更新 README.md。

README 必须包含以下结构：

### 项目说明
- 当前项目用途
- 技术架构（小游戏 / Canvas 等）

### 核心玩法
- 分层方块
- 暂存槽
- 收纳框
- 胜负规则

### 关键系统设计（必须突出）
- 可解关卡生成（solver）
- 生成 → 验证闭环流程

### 项目当前状态（必须更新）
分为三部分：
- 已完成
- 当前正在开发
- 下一步计划

### 运行方式
- 微信开发者工具说明

### 项目结构
- 各核心文件说明

---

## 2. 写作风格要求

- 使用中文
- 结构清晰，分段明确
- 不写废话，不写AI套话
- 必须突出“系统设计”，不是简单功能罗列
- 重点强调：
  - 可解性（solver）
  - 关卡生成逻辑
  - 游戏规则完整性

---

## 3. 提交规范

每次修改必须：

1. 更新代码
2. 更新 README（如果涉及功能/结构）
3. 提交 commit

commit message 要求：

- 使用 feat / fix / refactor 前缀
- 描述清楚做了什么

---

## 4. 禁止行为

- 不允许修改 README 结构为简单描述
- 不允许删除“项目当前状态”
- 不允许忽略 solver 说明
- 不允许生成“模板化 README”

---

## 5. 优先级原则

README 的作用是：

👉 让任何人一眼看懂项目进度和系统设计

高于代码注释的重要性
